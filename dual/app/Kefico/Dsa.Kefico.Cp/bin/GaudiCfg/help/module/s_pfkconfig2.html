   Beschreibung der Moduluebergabeparameter:
1 PRUEFSCHRITT
   	GAUDI PrÏfschritt
   ------------------------------
2 POSITION
   	PAV- Position
   ------------------------------ 
   Folgende Parameter stehen jeder fuer ein Prueftor (Prueftyp). Die Reihenfolge der Prüftore 
   wird durch die Eintraege in den folgenden Parametern festgelegt. 
   Z.B. PRUEFTOR1 = FO 
	PRUEFTOR2 = LT 
	Jetzt werden nur die Prueftore FO und LT erwartet, wobei FO als erstes Prueftor 
	definiert ist.
    
    ACHTUNG: Ist ein Prueftor nicht definiert, so kann die Pruefung an diesem 
	     Stand nicht durchgeführt werden.
3 PRUEFTOR1
  (Prueftyp) Unten genannt sind Beispiele
                     Der Prueftyp ist aus dem Prueflistennamen definiert. z.B. 
                     acep/01/1234567890 auf pruef_r_acep -> Prueftyp = EP 
		IC = In Circuit Tester		
		PF = Flashprogrammierung	(Programming Flash)
		FO = Funktionspruefung am offenen Geraet	(Function test Open ecu)
		LT = Tieftemperatur am geschlossenen Geraet	(Low Temperature funktion test)
		HT = Hochtempreatur am geschlossenen Geraet	(High Temperature funktion test)
		IR = Inline Run in
		FR = Endpruefung nach IRI oder Screening
		REP= REPair

    HINWEIS:
       Der Zusammenhang zwischen Standtyp und den oben genannten Prueftypen wird durch ein
       Configfile hergestellt, welches in jedem Werk verschieden sein kann und ebenso Produktspezifisch
       unterschiedlich sein kann. Kann kein zusammenhang im Conifg-File zwischen hier eingetragenem
       Prueftypen und einem Standtypen hergestellt werden, so ist eine PFK nicht moeglich.
       Dieses File wird im Ruestlauf vom Modul S_PFK2 eingelesen und interpretiert.
